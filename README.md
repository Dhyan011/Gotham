## 🛑 The Challenge: Moving Beyond Manual Records
The Karnataka State Police maintains extensive crime records capturing incidents, offenders, and victims. However, the current analytical ecosystem faces significant hurdles:
*   **Data Silos**: Highly normalized databases (e.g., the 23-table ER schema) trap data in silos, making network analysis nearly impossible.
*   **Reactive Policing**: Without systematic exploration of emerging trends, investigators lack the tools for proactive policing strategies.
*   **Lack of Advanced Analytics**: Crucial behavioral patterns, social interactions, and interconnected criminal networks remain undiscovered due to a lack of AI and Graph algorithms.

## 🚀 The Solution: GOTHAM
GOTHAM is an enterprise-scale intelligence platform that transforms raw FIR data into a proactive **Strategic Intelligence Hub**. By leveraging Graph Analytics, Generative AI, and Machine Learning, GOTHAM connects fragmented data points to reveal organized crime structures and predict emerging threats.

### 🌟 Key Capabilities
1. **Advanced Visualization**: Interactive Mapbox dashboards replace static Excel sheets. Spatiotemporal DBSCAN clustering identifies "Crime Hotspots" while CSS animations provide pulsing "Emerging Trend Alerts".
2. **Criminological Network & Link Analysis**: GOTHAM ingests relational data into **Neo4j** to visualize criminal associations (using Cytoscape.js). It uncovers hidden gang communities (Louvain Algorithm) and identifies influential brokers (PageRank & Betweenness Centrality).
3. **Sociological Dashboards**: Overlays crime data with urbanization patterns and economic indicators extracted from the 2025-26 Karnataka Economic Survey to understand the *"why"* behind the *"where"*.
4. **Predictive Risk & Anomaly Detection**: Utilizes **Isolation Forests** to flag incidents that deviate from standard behavioral patterns, and **XGBoost** for predictive risk scoring.
5. **Entity Resolution**: **RapidFuzz** algorithms (Jaro-Winkler) automatically merge duplicate or inconsistent accused records to build unified criminal profiles.
6. **Generative AI Copilot**: An LLM-powered assistant allows investigators to query the complex 23-table database in natural language and automatically generates intelligence summaries.

## 🛠️ Enterprise Architecture & Tech Stack

This project utilizes a highly scalable, real-world intelligence stack:

| Layer | Technologies | Purpose |
|-------|-------------|---------|
| **Streaming & Processing** | Apache Kafka, Apache Spark | Real-time ingestion and distributed batch processing of FIR data. |
| **Persistence (Source of Truth)** | PostgreSQL 15, PostGIS | Strict adherence to the official KSP 23-table ER schema; Geospatial data. |
| **Graph Intelligence** | Neo4j, Neo4j Graph Data Science (GDS) | Network modeling, Pathfinding (Dijkstra), Link Prediction (Node2Vec), Community Detection. |
| **Search Engine** | Elasticsearch 8 | Lightning-fast full-text and fuzzy searching across millions of records. |
| **Machine Learning** | scikit-learn, XGBoost, Prophet, RapidFuzz | Anomaly detection, risk scoring, time-series forecasting, and entity resolution. |
| **Backend API** | Python 3.11, FastAPI, SQLAlchemy | High-performance async REST API. |
| **Frontend & Viz** | React 18, Vite, TailwindCSS | "Dark Intelligence" glassmorphism aesthetic. |
| **Data Viz Libraries** | Mapbox GL, Apache ECharts, Cytoscape.js | High-performance interactive maps, charts, and graph networks. |

## 📊 Datasets
GOTHAM relies on official aggregate statistics to guide its highly realistic synthetic data generation engine:
*   `Karnataka Crime Data 2025.csv` (IPC & SLL distributions)
*   `Karnataka Vehicle Registrations Data.csv`
*   `Economic Survey 2025-26 English.pdf` (for sociological overlays)

*Note: All PII (Personally Identifiable Information) generated by the system is purely synthetic and modeled probabilistically off the aggregate real-world data distributions.*

## ⚙️ Quick Start Guide

### 1. Start the Infrastructure (Docker)
Ensure Docker is installed and running on your machine. This will spin up Postgres, Neo4j, Elasticsearch, and Kafka.
```bash
docker-compose up -d
```

### 2. Backend Setup
```bash
cd gotham-backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Generate synthetic data based on the 2025 CSV distributions
python -m ingestion.generate_data

# Start the API server
python main.py
```

### 3. Frontend Setup
```bash
cd ../gotham-frontend
npm install
npm run dev
```

*Access the GOTHAM Command Center at `http://localhost:5173`*
